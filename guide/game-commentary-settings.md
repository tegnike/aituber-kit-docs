# ゲーム実況モード設定

## 概要

ゲーム実況モードは、AIキャラクターがリアルタイムでゲーム実況を行う機能です。画面を定期的にキャプチャし、マルチモーダルAIモデルにスクリーンショットを送信して、感情付きの実況コメントを生成します。YouTubeライブ配信と組み合わせることで、視聴者とインタラクションしながらゲーム実況を行うことも可能です。

::: tip
ゲーム実況モードを使用するには、マルチモーダル対応（画像認識機能付き）のAIモデルを設定する必要があります。
:::

## ゲーム実況モード

ゲーム実況モードの有効/無効を切り替えます。有効にすると、メインページにゲームコントローラーボタンが表示されます。

::: warning 排他制御
ゲーム実況モードは以下の機能と排他制御されています。ゲーム実況モードを有効にすると、これらの機能は自動的に無効になります。

- アイドルモード
- 人感検知
- リアルタイムAPI
- オーディオモード
- 外部連携モード

なお、YouTubeモードとは排他ではなく、同時に利用できます。
:::

**環境変数**:

```bash
# ゲーム実況モードの有効/無効（true/false）
NEXT_PUBLIC_GAME_COMMENTARY_ENABLED="false"
```

## 使い方

1. 設定画面でゲーム実況モードを有効にします
2. メインページにゲームコントローラーボタンが表示されます
3. ボタンをクリックすると実況が開始されます（画面キャプチャが自動的に始まります）
4. AIが画面をキャプチャ → 実況コメントを生成 → 音声で読み上げ → 待機 → 繰り返し
5. もう一度ボタンをクリックすると実況が停止します

## キャプチャ間隔

画面キャプチャの間隔を秒単位で設定します（10〜60秒）。前回の実況コメントの生成・読み上げが完了してから次のキャプチャまでの待機時間です。コメントの重複を防ぐため、setIntervalではなくsetTimeoutベースで制御されています。

**環境変数**:

```bash
# キャプチャ間隔（秒）
NEXT_PUBLIC_GAME_COMMENTARY_CAPTURE_INTERVAL="15"
```

## 参照コンテキスト数

実況コメント生成時に参照する過去の実況コメント数を設定します（1〜20）。過去のコメントをコンテキストとして含めることで、同じ内容の繰り返しを防ぎ、より自然な実況を実現します。実況履歴はチャットログとは別のリングバッファで管理されます。

**環境変数**:

```bash
# 参照コンテキスト数
NEXT_PUBLIC_GAME_COMMENTARY_CONTEXT_COUNT="5"
```

## プロンプトテンプレート

実況コメント生成に使用するプロンプトテンプレートをカスタマイズできます。テンプレートを空にするとデフォルトのプロンプトが使用されます。

**環境変数**:

```bash
# 実況プロンプトテンプレート
NEXT_PUBLIC_GAME_COMMENTARY_PROMPT_TEMPLATE=""
```

## 画像品質

キャプチャ画像のJPEG品質を設定します（0.3〜1.0）。値が低いほどファイルサイズが小さくなり、API呼び出しのコストを抑えられますが、画質が低下します。

**環境変数**:

```bash
# キャプチャ画像品質（0.3-1.0）
NEXT_PUBLIC_GAME_COMMENTARY_IMAGE_QUALITY="0.7"
```

## リサイズ幅

キャプチャ画像のリサイズ幅をピクセル単位で設定します。0を指定するとリサイズなしになります。画像を小さくすることでAPI呼び出しのコストを抑えられます。

**環境変数**:

```bash
# リサイズ幅（px、0=リサイズなし）
NEXT_PUBLIC_GAME_COMMENTARY_RESIZE_WIDTH="1024"
```

## チャットログへの保存

実況コメントをチャットログに保存するかどうかを設定します。有効にすると、実況コメントが `[実況]` プレフィックス付きでチャットログに保存されます。これにより、YouTube連携やMastraワークフローなど、チャットログを参照する他の機能とコンテキストを共有できます。

**環境変数**:

```bash
# チャットログへの保存（true/false）
NEXT_PUBLIC_GAME_COMMENTARY_SAVE_TO_CHAT="true"
```
